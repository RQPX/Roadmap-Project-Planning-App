"Design a comprehensive, professional Project Management Web Application Dashboard optimized for desktop use. The primary user is a Technical Project Manager managing a portfolio of 75+ projects, requiring full CRUD (Create, Read, Update, Delete) capabilities. The secondary users are Directors who need a read-only overview. The aesthetic must be clean, light background, using professional blue accent colors and highly specific status colors. Use Autolayout strictly for all components. All text labels must be in French. Include realistic sample data for an IT and Technical enterprise context (e.g., 'Déploiement Serveurs', 'Intégration API').
Global Rules & Colors:
Status Colors (Strict): En exécution (Yellow/Beige), En étude (Purple), Clôturé (Light Blue), En attente de Go pour production (Light Green), En production (Dark Green), En pause (Gray), Non démarré (Gray-blue), Abandonné (Deep Red), En service (Bright Green).
Priority Colors: Haute (Red), Moyenne (Orange), Basse (Gray).
Navigation: A persistent top navigation bar with: 'Tableau de Bord', ' Projets', 'Chronologie'. Include a User Profile avatar with the name 'Admin Projets'.
Screen 1: Tableau de Bord (Dashboard Overview)
Header: Title 'Vue d'ensemble des Projets'.
KPI Cards (Top row): 3 large cards showing 'Projets Actifs' (Number: 42), 'Projets en Retard' (Number: 5, use red warning icon), and 'Taux d'Avancement Moyen' (Progress bar showing 68%).
Chart Section: A large Pie Chart titled 'Répartition par Statut' showing the distribution of projects using the exact status colors defined above.
List Section: A wide table titled 'Récemment Mis à Jour'. Columns: Nom du Projet, Département (e.g., DSI, DGP, DAF), Statut (colored badge), Avancement (progress bar), Date de Fin Prévue.
Screen 2: Projets (Project Management Board)
Header: Title 'Gestion des Projets'. Include a prominent primary button '+ Ajouter un Nouveau Projet'.
Kanban Board: A horizontal scrollable area with columns for every status defined above.
Kanban Cards: Each project card must show the Project Name, a Department badge (e.g., [DSI]), Priority badge, a 0-100% progress bar, the Project Manager's name, and a red 'RETARD' alert badge if applicable.
Screen 3: Formulaire d'Ajout/Édition de Projet (Modal/Overlay) (when clicking on the project. must be able to edit or delete a project in his card)
Create a detailed overlay or slide-out panel for creating or editing a project.
Header: 'Détails du Projet' with 'Enregistrer' (Save) and 'Supprimer le projet' (Delete - text in red) buttons.
Form Fields (clearly labeled in French):
Nom du Projet (Text input)
Département (Dropdown: DSI, DAF, RH, DGP, DDEV)
Statut (Dropdown with the 9 specific status colors)
Priorité (Dropdown: Basse, Moyenne, Haute)
Chef de Projet (Text input)
Partenaires / Équipe (Text input)
Date de Début (Date picker input)
Date de Fin Prévue (Date picker input)
Taux d'Avancement (Number input / Slider 0-100%)
Commentaires et Mises à jour (Large text area).
(Screen 3 must be when we open a project kanban card, we must see all these informations above perfectly layed out)
Screen 4: Chronologie (Gantt Chart Timeline)
Header: Title 'Chronogramme Global'. Include filters for 'Département' and 'Statut'.
Left panel: A vertical list of projects categorized by their Department sigil.
Right panel: A horizontal timeline spanning weeks. Display continuous colored bars for each project duration, with the bar color matching its current status color."
